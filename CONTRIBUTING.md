# Contributing to Vue Development Skill

Thank you for your interest in contributing! This guide will help you get started.

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest improvements
- Provide clear examples and context
- Include relevant code snippets or screenshots

### Suggesting Enhancements

We welcome suggestions for:
- Additional patterns and best practices
- New templates
- Real-world examples
- Documentation improvements
- Performance optimizations

### Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Ensure documentation is updated
5. Commit with clear messages: `git commit -m "Add feature: description"`
6. Push to your fork: `git push origin feature/your-feature-name`
7. Open a Pull Request

## Documentation Standards

### Writing Style

- Be clear and concise
- Use code examples extensively

- Include "Why" and "When to use" sections
- Keep examples practical and realistic

### Code Examples

- Use TypeScript
- Follow Vue 3 Composition API style
- Include comments for complex logic
- Show complete, runnable examples when possible

### Structure

Each pattern should include:
1. **Problem** - What issue does this solve?
2. **Solution** - The pattern or approach
3. **Benefits** - Why use this pattern?
4. **Trade-offs** - When NOT to use it
5. **Example** - Working code
6. **Testing** - How to test it

## Adding New Templates

Templates should:
- Follow the existing structure
- Include comprehensive comments
- Show best practices
- Be immediately usable
- Include usage examples

## Code of Conduct

### Our Standards

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Assume good intentions
- Help others learn

### Not Acceptable

- Harassment or discrimination
- Trolling or insulting comments
- Publishing others' private information
- Any unprofessional conduct
